# Дизайн: ввод значений показателя выше максимума в отчёте

**Дата:** 2026-04-24
**Проект:** Ибадат Трекер (reportdeepen)
**Затрагиваемые экраны:** Отчёт (редактор), Детали, Главная
**Основной файл:** [lib/screens/report/report_editor_screen.dart](../../../lib/screens/report/report_editor_screen.dart)

## Цель

Позволить пользователю (и администратору по своим личным показателям) указывать в отчёте значения выше максимума, заданного во вкладке «Админ». Сейчас UI рассинхронизирован: слайдер клампится до `metric.maxValue`, а кнопка «+» потолка не имеет — значение уходит выше, слайдер «залипает» на максимуме, и числа выглядят противоречиво.

## Поведение (вариант C — комбинированный)

1. **Слайдер** работает в диапазоне `0..maxValue` и визуально клампится — без изменений.
2. **Кнопки «−» / «+»** — как сейчас: «−» блокируется на 0, «+» без верхнего потолка.
3. **Быстрые значения 25/50/75/100% от max** — без изменений.
4. **Тап по центральной цифре** открывает диалог ручного ввода `_ManualValueDialog`:
   - `TextField` с предзаполненным текущим значением, числовая клавиатура, `FilteringTextInputFormatter.digitsOnly`.
   - Подсказка под полем: «Обычный диапазон: 0..{max} {unit}».
   - Валидация: только неотрицательные целые. Пустое или невалидное значение → кнопка «Сохранить» неактивна.
   - Кнопки: «Отмена» / «Сохранить» (цвет `metric.color`).
5. **Бейдж переполнения:** при `value > metric.maxValue` рядом с чипом-значением показывается маленький жёлтый бейдж `+{value - max}`.

## Поток данных

```
Тап на чип цифры
  → showDialog<int>(_ManualValueDialog(current, max, unitLabel, color))
  → pop(int newValue)
  → setState(_report.setValue(metricId, newValue))
  → UI обновляется: слайдер клампится до max, чип показывает raw value, бейдж "+N" появляется если value > max
  → _save() → _repo.upsertReport(_report) → jsonb metric_values
```

## Граничные случаи

- `metric.maxValue == 0` — в теории невозможно: валидация в форме админа ([admin_screen.dart:3715](../../../lib/screens/admin/admin_screen.dart#L3715)) отсекает `maxValue <= 0`. На всякий случай `sliderMax = 1.0` уже защищает слайдер; бейдж переполнения также корректно не появится.
- Пустая строка в поле ввода → «Сохранить» неактивна.
- Значение = 0 через ручной ввод → валидно (пользователь может сбросить).
- Значение ровно `maxValue` → бейджа нет.

## Тесты

- Существующие тесты `test/reporting/report_progress_test.dart` не затрагиваются.
- Widget-тест (опционально, вне MVP): тап по чипу открывает диалог; ввод значения > max сохраняет его и показывает бейдж `+N`.
- Ручная проверка: и обычный пользователь в группе, и админ на личных показателях — оба сценария используют один `_buildMetricCard`, работает симметрично.

## Что не меняется

- Логика расчёта процента (`metricProgress`, `reportProgress`) — уже клампит до 1.0.
- Слайдер — остаётся «шкалой нормы» 0..max.
- Быстрые значения — остаются 25/50/75/100% от max.
- Структура БД, миграции — не требуются.
